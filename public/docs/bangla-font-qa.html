বাংলা ফন্ট — মাল্টি-Viewer QA গাইড

এই গাইডটি পরীক্ষা করে দেখায় যে আপনার তৈরি PDF গুলোতে বাংলা টেক্সট Chrome, Adobe Reader, এবং মোবাইল PDF অ্যাপে একইভাবে রেন্ডার হচ্ছে কিনা। সব PDF-এ ফন্ট subset-embed হয়, তাই viewer-এ ফন্ট ইনস্টল না থাকলেও কাজ করার কথা।

১. দ্রুত যাচাই (৫ মিনিট)

  1. Settings → বাংলা ফন্ট কার্ডে যান।
  2. "QA নমুনা PDF ডাউনলোড" বাটনে ক্লিক করুন।
  3. নিচের viewer তিনটিতে একই PDF খুলে নিচের checklist মিলিয়ে দেখুন।

২. Viewer-ভিত্তিক টেস্ট ম্যাট্রিক্স

Viewerপ্ল্যাটফর্মকীভাবে যাচাই করবেনসম্ভাব্য সমস্যা
Chrome / Edge
(built-in)		Desktop PDF ফাইল ব্রাউজারে drag করুন। Ctrl+ +/- দিয়ে zoom করে যুক্তাক্ষর দেখুন। সাধারণত কোনো সমস্যা হয় না (HarfBuzz shaping)।
Adobe Acrobat ReaderDesktop File → Open → আপনার PDF। Edit → Preferences → Page Display-এ "Use local fonts" off রাখুন। পুরনো version-এ কখনো কখনো reph/matra position হালকা সরে যায় — font subset embed-এ সাধারণত ঠিক থাকে।
Firefox PDF.jsDesktop PDF Firefox-এ open করুন। About:config-এ pdfjs.disabled false থাকা দরকার। PDF.js জটিল conjunct-এ মাঝে মাঝে gap দেখায় — embed font থাকলে minimal।
Adobe Reader MobileAndroid / iOS Play/App Store থেকে install করে PDF open করুন। খুব কম। রিপোর্ট করুন যদি hashma/reph ভেঙে যায়।
Google Drive PreviewMobile + Web Drive-এ upload করে in-app preview খুলুন। Drive web preview কখনো কখনো embedded font ignore করে — Drive অ্যাপে ঠিক থাকে।
Samsung / Xiaomi Default ReaderAndroid Default file manager → PDF tap। OEM viewer-এ কখনো কখনো glyph fallback হয় — Adobe Reader recommend করুন।
iOS Files / BooksiPhone / iPad AirDrop বা mail attachment → tap to open। iOS-এর CoreText shaping ভালো — সাধারণত নিখুঁত।

৩. Checklist (প্রতিটি viewer-এ মিলান)

  • "কৃষ্ণচূড়া" — ক্ষ্ণ যুক্তাক্ষর সঠিকভাবে গঠিত (ভেঙে আলাদা নয়)
  • "স্ত্র" — তিনটি অক্ষর একসাথে stack হয়েছে
  • "কৃষক" — কৃ-র নিচে হ্রস্ব ঋ-কার ঠিক জায়গায়
  • "০ ১ ২ ৩ ৪ ৫ ৬ ৭ ৮ ৯" — সব বাংলা সংখ্যা পড়া যাচ্ছে (□ box নয়)
  • দাঁড়ি (।) সঠিকভাবে রেন্ডার হয়েছে
  • matra (ি, ী, ে, ৈ, ো, ৌ) ঠিক position-এ বসেছে
  • reph (র্) যুক্তাক্ষরের উপরে ছোট হয়ে বসেছে
  • Latin শব্দ (Loan, SMS, NID) আলাদাভাবে ঠিকঠাক — overlap নেই

৪. সমস্যা হলে (Fallback Rules)

লক্ষণসম্ভাব্য কারণসমাধান
সব বাংলা □ boxFont load fail হয়েছে।App স্বয়ংক্রিয়ভাবে fallback ফন্টে যাবে। Browser console-এ [pdf] Bangla font ... failed warning দেখুন।
যুক্তাক্ষর ভেঙে যাচ্ছেViewer পুরনো / shaping সমর্থন নেই।Viewer update করুন বা Adobe Reader/Chrome ব্যবহার করুন।
Latin অংশ অদৃশ্যএকই PDF object-এ font mix সমস্যা।Settings → বাংলা ফন্ট → অন্য ফন্ট বাছুন (Sans ↔ Serif)।
প্রিন্টে ঝাপসাLow-DPI scaling।Print dialog-এ "Actual Size" / 100% scale বাছুন।

৫. Automatic Fallback — কীভাবে কাজ করে

প্রতিবার PDF তৈরির সময় ensureBanglaFont(doc) কল হয়। এটি:

  1. আপনার পছন্দের ফন্ট fetch করার চেষ্টা করে।
  2. ব্যর্থ হলে BANGLA_FONT_FALLBACK-এর পরের ফন্ট চেষ্টা করে।
  3. সফল হলে সেই family-নাম return করে এবং PDF-এ subset-embed করে।
  4. সব ব্যর্থ হলে null return করে — কলার তখন default Helvetica-তে fallback করে এবং console-এ warning log হয়।
